### 3. 新增状态管理函数

#### ResetDeploymentState(deploymentID string) error
- 功能：重置 deployment 状态
- 清除字段：
  - exhausted_until (设为 nil)
  - cooldown_until (设为 nil)
  - last_error_code (设为空字符串)
  - last_error_message (设为空字符串)
- **不**清除：
  - used_prompt_tokens
  - used_completion_tokens
  - used_total_tokens
  - request_count
  - success_count
  - error_count
- 用途：手动恢复被标记为 exhausted 的 deployment

#### ClearDeploymentExhausted(deploymentID string) error
- 功能：仅清除 exhausted_until
- 用途：快速恢复 exhausted 状态

#### ClearDeploymentCooldown(deploymentID string) error
- 功能：仅清除 cooldown_until
- 用途：快速恢复 cooldown 状态

#### GetAllDeploymentStates(virtualModel string) (map[string]*DeploymentState, error)
- 功能：获取虚拟模型下所有 deployment 的状态
- 返回完整的 DeploymentState 信息

#### 修改文件
- `fallback/state.go:216-265` - 新增上述函数

### 4. 新增配置重载功能

#### ReloadConfig(path string) error
- 功能：从指定路径重新加载配置
- 锁保护：使用 configLock 确保线程安全
- 验证流程：
  1. 读取并解析新配置文件
  2. 验证新配置的有效性
  3. 如果验证通过，更新全局 config
  4. 记录日志
- 安全性：
  - reload 失败不会覆盖旧配置
  - 先验证再更新
  - 使用写锁确保原子性

#### ValidateConfigWithLock() error
- 功能：在锁保护下验证配置
- 验证内容：
  - config 是否已加载
  - fallback 是否启用
  - 是否有虚拟模型配置
  - 每个虚拟模型的 fallback_order 是否完整
  - 每个引用的 deployment 是否存在

#### loadConfigFromFile(path string) (*Config, error)
- 功能：内部函数，加载并解析配置文件
- 处理默认值：
  - SoftLimitRatio 默认 0.9
  - HardLimitRatio 默认 0.98

#### 修改文件
- `fallback/config.go:14` - 添加 time 导入
- `fallback/config.go:268-324` - 新增上述函数

#### POST /api/fallback/deployments/:id/reset
- 功能：重置 deployment 状态
- 权限：管理员
- 参数：id (deployment ID，如 "primary")
- 响应：
```json
{
  "message": "deployment state reset successfully"
}
```

#### POST /api/fallback/deployments/:id/clear-exhausted
- 功能：仅清除 exhausted_until
- 权限：管理员
- 参数：id (deployment ID)
- 响应：
```json
{
  "message": "deployment exhausted state cleared successfully"
}
```

#### POST /api/fallback/deployments/:id/clear-cooldown
- 功能：仅清除 cooldown_until
- 权限：管理员
- 参数：id (deployment ID)
- 响应：

- 功能：重新加载配置文件
- 权限：管理员
- 查询参数：path (可选，默认 "data/fallback.json")
- 响应：
```json

  "message": "configuration reloaded successfully"
}
```
